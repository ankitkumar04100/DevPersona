# Contributing to DevPersona

Thank you for your interest in contributing to **DevPersona: Interactive Portfolio Experience**! This guide explains how you can help improve the project, report issues, or add features in a structured and professional way.

---

## âœ… How to Contribute

You can contribute in multiple ways:

1. **Bug Reports / Issues**  
   - If you find any bugs, please create a GitHub issue with:  
     - **Title**: Short summary of the bug  
     - **Description**: Steps to reproduce, expected behavior, actual behavior  
     - **Screenshots**: Optional but helpful  

2. **Feature Requests**  
   - Have an idea to improve DevPersona? Create a GitHub issue with:  
     - **Title**: Feature name  
     - **Description**: Detailed explanation of the feature and its purpose  
     - **Examples / Mockups**: Optional sketches or screenshots  

3. **Code Contributions**  
   - Fork the repository and create a new branch for your feature or fix:  
     ```bash
     git checkout -b feature/your-feature-name
     ```
   - Make your changes, following the **existing code style** (React, Tailwind, modular components)  
   - Test your changes locally  
   - Commit with clear messages:
     ```bash
     git commit -m "Add [feature/fix]: short description"
     ```
   - Push your branch and create a **pull request**
